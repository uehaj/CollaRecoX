2. server.js — OpenAI realtime接続の撤去（外科的）

設計判断: /api/realtime-ws はOpenAIプロキシ専用ではなく、 オンデバイス認識結果・自動校正を共有ドキュメントへ中継する主経路でもある（local_transcription/set_auto_proofread/local_pending 等）。 そのため connectionハンドラ丸ごと削除は破壊的。OpenAIへの実接続だけを断つ最小影響方針を採用。

撤去（L293-607のopenaiWsイベントハンドラ群を削除）

  • new WebSocket('wss://api.openai.com/v1/realtime') 実接続 → 接続を張らないダミー(no-op, readyState=CLOSED) に置換
  • openaiWs.on('open'|'message'|'error'|'close') 4ハンドラ（OpenAI→client/Hocuspocus転送・session設定・"Connected to OpenAI"通知）を削除

温存（無傷を確認）

  • Hocuspocus初期化・/api/yjs-ws・/api/yjs-sessions・HMR・HTTPサーバ起動
  • 中継/校正関数 sendTextToHocuspocusDocument(L1225)・setPendingText・clearPendingText・maybeAutoProofread(L1797) — いずれも削除ブロック外で定義・中継ケースから呼び出し
  • OPENAI_API_KEYチェック・自動校正/AI編集の callChatCompletion（OpenAI chat completions）

残存する旧OpenAIプロキシのclientメッセージcase（set_prompt/audio_chunk 等）はクライアントが送らなくなったため到達不能なデッドコードであり、ダミーへのno-opとして無害化。深掘り削除は中継ケースと交錯するため最小影響を優先し見送り（要望あれば別タスクで整理可能）。バックアップ: /tmp/collarecox-removed-20260613/server.js.before-openai-surgery。
